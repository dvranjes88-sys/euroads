# 🚢 Ferry Integration Feature — Spec

## Vision
Make EuroRoads the **first travel site that gives complete trip cost including ferries**, with departure times, booking links, and reverse departure planning. Killer feature for HR coastal trips and Stockholm→DE corridor.

Example user flow:
```
User: Bratislava → Hvar (Croatian island)
EuroRoads:
  🚗 Drive: Bratislava → Split (8h 30m, €87 toll+fuel)
  🚢 Ferry: Split → Stari Grad (Hvar)
     ⏰ Next: 16:30 (Today)
     💶 Car ≤5m + 2 adults: €52.40
     ⏱️  2h crossing
     📍 Arrive Stari Grad: 18:30
  
  ⚠️  Suggested departure: Leave Bratislava by 7:30
      (30 min ferry buffer + drive time)
  
  💰 Total: €139.40
  🎟️ [Book ferry on Jadrolinija →]
```

## Why this matters strategically
- **No competitor has this** — Google Maps shows "ferry available" without prices/times. ViaMichelin ignores ferries. Direct Ferries shows ferries but no driving cost.
- **HR-specific killer feature** — primary EuroRoads audience (SK/CZ → HR coastal) often needs island ferries.
- **Trip cost honesty** — Storebælt Bridge €34, Öresund €67 are massive missing items today.
- **Trust building** — accurate end-to-end planning = repeat users.

## Phase Plan

### **Phase 1: MVP — Detection + Static Pricing** (2-3 hours)
Goal: User sees ferry exists in route + estimated price. Foundation for V2.

**Scope:**
1. Detect ferry segments in Google Directions response (parse `route.legs[].steps[]` for `travel_mode === 'FERRY'` or instructions containing "ferry"/"Take the boat")
2. Hardcoded JSON with top 10 ferries (Croatia internal + Storebælt + Öresund + Italy-Croatia)
3. Display new row in cost breakdown: `🚢 Ferry: Split → Hvar · ~€52 · book on Jadrolinija →`
4. Highlight ferry segment on map (purple/blue color, distinct from road)
5. Booking link to operator's site

**Out of scope for V1:**
- ❌ Live timetables
- ❌ Departure time suggestions
- ❌ Price scraping automation
- ❌ Multi-leg routing optimization

**Success criteria:**
- Stockholm → DE route shows Storebælt €34 in breakdown
- Split → Hvar route shows ferry segment + estimated cost
- Booking link opens correct operator page

---

### **Phase 2: Live timetables + Booking links** (1-2 days)
Goal: Real ferry times, accurate prices via API, deep booking links with VoyageId.

**Scope:**
1. Reverse engineer Jadrolinija API (`POST https://www.jadrolinija.hr/ajax/price-list.php`)
2. GitHub Action nightly scraper (similar to fuel-prices pipeline):
   - Fetches prices for top 30 routes
   - Generates `ferry-prices.json` 
   - Auto-commits if changed
3. For Storebælt/Öresund: scrape from Sund & Bælt + Øresundsbron sites
4. Display: `🚢 Next ferry: Split → Hvar · Today 16:30 · €52 · 2h crossing`
5. Booking URL with `VoyageId` parameter (deep link)

---

### **Phase 3: Reverse departure planning** (1 week)
Goal: User picks "I need to arrive by X", system calculates everything backwards.

**Scope:**
1. Add **"Departure time" picker** in route input UI
2. Logic: target arrival → ferry → port buffer (30 min) → drive time → origin departure
3. Display "Suggested departure: Leave by HH:MM" banner
4. Multi-leg breakdown with timestamps per stage
5. Warning system: "Last ferry today is at 22:00, you can't make it"

## Technical Architecture

### Data sources

**Jadrolinija (Croatia):**
- Endpoint: `POST https://www.jadrolinija.hr/ajax/price-list.php`
- Response: HTML wrapped in JSON `{status: "success", content: "<HTML>"}`
- Port codes: 3-letter (BIO=Biograd, TKO=Tkon, SUP=Supetar, STG=Stari Grad, etc.)
- Pricing categories: Adult, Children 3-12, Children 1-3, Baby (0-1), Car ≤5m, Car >5m, Camper, Motorcycle, Bicycle, Bus, Lorry, Trailer
- VAT 25% included in displayed prices
- **CORS issue:** Browser-side fetch blocked → must scrape via GitHub Action backend

**Storebælt Bridge (Denmark):**
- URL: https://www.storebaelt.dk/priser
- Static pricing (changes ~1× per year)
- ~€34 one-way for car ≤6m

**Øresund Bridge (DK ↔ SE):**
- URL: https://www.oresundsbron.com/en/prices
- ~€67 one-way for car

**Scandlines (Puttgarden ↔ Rødby):**
- URL: https://www.scandlines.com/prices
- €60-90 one-way for car (variable by season)

**Google Directions API:**
- Already used. Default includes ferries.
- Detect ferry segments in `route.legs[].steps[]`:
  - Check `travel_mode === 'FERRY'` OR `instructions.includes('ferry')` OR `instructions.includes('boat')`
- Extract: departure port lat/lng, arrival port lat/lng, distance, duration

## Edge cases to handle

1. **Multiple ferry options per route** — Drvenik→Sućuraj vs Split→Stari Grad both reach Hvar. Show user choice.
2. **Off-season schedules** — Summer has 8 daily, winter 2 daily. Need date-aware lookup.
3. **Ferry full / sold out** — Out of scope for MVP, mention "advance booking recommended" in summer months.
4. **Walking-only ferries** — Some routes are passenger-only (catamaran). Note "passenger only, leave car at port".
5. **Pre-2000 historical data** — Some scraped sites might return outdated info. Always check `last_updated` timestamp in JSON, show stale warning if >2 weeks old.
6. **Currency conversion** — All prices in EUR (HR is Eurozone since 2023). Storebælt/Øresund are DKK → convert.
7. **Vehicle category mapping** — User selects "Car" → match to ferry's "car_under_5m" by default. Allow override.
8. **Last-leg edge case** — What if route ends ON an island and starts ON an island? (Hvar → Split → Brač). Phase 2.
9. **Ferry not detected** — Google Directions sometimes routes around ferries. Add UI hint: "Include ferries: ✓"
10. **Booking link expiry** — Some operators have time-limited URLs. Always generate fresh link with current date.

## Decisions needed (open questions)

1. **Where to put ferry detection in render flow?** Should it be parallel to country detection or after?
2. **Show all ferry options or just cheapest?** If route has multiple, default to cheapest with toggle to compare.
3. **Booking affiliate program?** Check if Jadrolinija/Scandlines have affiliate program for revenue.
4. **When to mark route as "ferry-required"?** Some routes have alternative (Drvenik→Sućuraj) but driver might prefer the toll-free direct ferry. Show choice.
5. **Pricing categories complexity** — Jadrolinija has 25+ categories. For MVP show only "Car ≤5m + Adult driver". Phase 2 add passenger picker.
