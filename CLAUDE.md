# EuroRoads — Project Context for Claude Code

## What this is
EuroRoads (https://euroads.app, https://euroads.vercel.app) is a single-page European driving reference site. Users can:
- Browse country-by-country toll/vignette/speed/environmental zone info (5 tabs)
- Calculate route costs (toll + fuel + vignettes) across multiple EU countries
- Play a capitals quiz game

Built by a finance professional who code via "vibe coding" with Claude — not a classic developer. Keep code readable and well-commented (Slovak comments are fine, mixed with English is OK).

## Stack
- **Single HTML file** (`index.html`, ~4500 lines) — no build step, no npm dependencies for runtime
- **Mapbox GL JS** for maps (token in `_ta+_tb` split for code obfuscation)
- **Google Directions API** for routing (key in `_ga+_gb` split)
- **Vercel** for hosting + auto-deploy from GitHub on every commit
- **GitHub Actions** for nightly fuel price scraping (Node.js)

## API Keys (security)
- Both Mapbox + Google keys are **referrer-restricted** to `euroads.vercel.app/*` and `euroads.app/*`
- Keys are split in code as `_ta+_tb` (Mapbox) and `_ga+_gb` (Google) — concatenated at runtime
- Test files like `test-hr-coverage-v2.html` must be deployed (not opened locally) to use the keys

## Key features in index.html

### 1. Tabs (top navigation)
- `countries` — country cards with toll/vignette/fuel info
- `speed` — speed limit table per country
- `enc` — environmental zones (LEZ in Germany, etc.)
- `game` — European capitals quiz
- `maproute` — route calculator with cost breakdown

### 2. Route calculator (`maproute` tab)
The core feature. Flow:
1. User enters waypoints (geocoded via Mapbox)
2. Google Directions returns 3 alternatives (Fastest, Cheapest, Shortest)
3. For each route: detect countries via Mapbox reverse geocoding samples
4. Calculate costs per country:
   - **HR**: Live HAC API (`cro3-tmp.hac.hr/tollcalculator/price/...`) — multi-segment with open toll system handling
   - **AT, SI, CH, CZ, SK, HU, RO, BG**: Vignette-based (static prices)
   - **IT, FR, ES, PT, GR, PL**: Distance-based toll estimates
   - **DE, SE, DK, NO, FI**: Free for cars (motorways)
5. Add fuel cost per country (from `fuel-prices.json` or fallback)
6. Display in left panel + map color coding

### 3. HR-specific features (recently improved May 2026)
- **Live HAC API** integration with 3 fixes:
  1. **Dubrovnik southern endpoints** — Karamatići/Ravča/Vrgorac for routes ending lat <43.5 (instead of Karasovići which is MNE border)
  2. **Multi-border entry candidates** — tries up to 4 entry stations (mapped + 3 nearest to first HR points)
  3. **Dynamic NUM_SAMPLES** — 50/60/80/100 based on route length (so Stockholm→Rijeka has enough samples)
- **HR Live banner** — shows per-company breakdown of fuel prices (AdriaOil, INA, Lukoil, Petrol, Shell, Tifon)
- Tested with 50 routes (5 origins × 10 HR destinations) — 100% Live HAC coverage in production

### 4. Open toll system handling (HAC quirk)
HAC API returns segments like `Bregana → Bregana: €1.00` for A3 motorway entry fee (open toll system, not a bug). Code displays this as friendly label "Bregana entry fee" with explanation tooltip.

## Conventions
- **Slovak/English mix** is fine in code comments
- **No build step** — everything must work as static HTML
- **No npm dependencies for runtime** — only for GitHub Actions scripts
- **Inline fallbacks** for fetched data (e.g., fuel-prices.json has inline backup in HTML)
- **Test changes locally** by opening `index.html` in browser (file://) — but routing won't work without deployment due to API key referrer restrictions

## Deploy workflow
1. Make changes to files
2. `git add .` + `git commit -m "..."` + `git push`
3. Vercel auto-deploys from `main` branch (~30 seconds)
4. Hard refresh `euroads.app` to verify (Ctrl+Shift+R)
